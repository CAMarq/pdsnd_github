Question 1¶

What is the most common month?

#To Generate "Date" type column and assign name to Month

Month <- all_cities %>%
  mutate(Clean.Start.Date = as.Date(Start_Time),
         Clean.End.Time = as.Date(End_Time)) %>%
  mutate(Month = month(Clean.Start.Date)) %>%
  mutate(Month = case_when(Month == "1" ~ "January",
                         Month == "2" ~ "February",
                         Month == "3" ~ "March",
                         Month == "4" ~ "April",
                         Month == "5" ~ "May",
                         Month == "6" ~ "June"))


#To change column Month to be a factor

Month$Month <- factor(Month$Month, levels=month.name)

To obtain this result the first task was to take the Start Time & End Time and transform tell R that this is a date using the as.Date function, once it is converted to "Date" time we can do transformations, I used the library(lubridate) that gives us functions that separate dates depending on the analysis we want to do (by day, month, year, etc.). I separated per month to answer our question, the output will be the number of the month. Using the function mutate() + case_when() we can create "tags", for each month (number), to transform each row on the data.frame to its corresponding month. Once we have our "Month" column created with each month's full name, we need to change the column into a factor so later on ggplot2 we can use the function scale_x_discrete() and it'll recognize it to be ordered in calendar sequence instead of alphabetically. Since we're dealing with recurrences, a simple bar chart would be able to give us this information. Since our categorical data is Months, I assigned the x-axis to this variable "Month", and the count() function that will be used to create the plot by the package will be the aggregating the rows by this category, giving us the "Count of Users"

#Graph
all_cities %>%
  group_by(Start_Station) %>%
  summarise(Incidences=n()) %>%
  top_n(3) %>%
  ggplot(., aes(x=Start_Station, y=Incidences))+
  geom_bar(stat="Identity", color="orange", fill="orange") +
  xlab('Start Station') +
  ylab('Incidences') +
  ggtitle("Top 3 Start Stations")

Selecting by Incidences

The start station with the highest incidence is Columbus Circle/Union Station.

To be able to tell which was the start station that had the highest incidence, a bit of data wrangling to prepare our data is required. I performed a grouping per Start_Station to aggregate by unique station names, the summarise() + n() functions in conjunction give us a countdown of the individual rows that were aggregated and give that column a name, which I named "Incidences", if we left the data as is and create a graph out of this dataset we'd get graph that is impossible to read due to the ammount of Stations in the analysis. To avoid this I also added the top_n() function to give me top 3 Stations. Since the output will be correct but untidy, I used the arrange() +desc() functions to get an ordered result.

To build the graph I took the same logic I used to create our summary data explained previously, this is a way of subsetting our data frames to get the countdowns and aggregations needed and graph without having to create separate data tables. In reality, it wasn't necessary to create the Start_Stations data.frame, but it is useful to be able to reassure we're getting the correct calculations. The x-axis is the categorical data that will define the countdown that will be made by the plot (Incidences).
